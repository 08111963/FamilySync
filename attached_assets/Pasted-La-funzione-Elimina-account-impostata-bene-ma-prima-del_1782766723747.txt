La funzione “Elimina account” è impostata bene, ma prima dell’approvazione servono alcune correzioni.

Correggi questi punti:

1. Migration database obbligatoria

Hai aggiunto `users.deletedAt` / colonna `deleted_at`, ma nello ZIP non vedo migration.

Aggiungi una migration sicura per produzione:

ALTER TABLE users ADD COLUMN IF NOT EXISTS deleted_at timestamp;

Oppure usa il sistema Drizzle già previsto dal progetto, ma deve essere chiaro quale migration verrà applicata al database reale.

Senza questa migration il backend può andare in errore quando legge `users.deletedAt`.

2. AuthGate: consentire delete-account anche a email non verificata

Nel backend hai correttamente permesso la cancellazione account anche a utenti con email non verificata.

Però in `app/_layout.tsx`, AuthGate rimanda gli utenti non verificati alla schermata `verify-email`.

Aggiungi `delete-account` tra le schermate consentite anche se l’email non è verificata.

Obiettivo:

* utente autenticato ma non verificato deve poter aprire `/delete-account`;
* deve poter cancellare l’account;
* poi logout automatico.

3. Cancellazione file fisici caricati

Quando l’utente è unico membro di una famiglia e la famiglia viene eliminata, il DB cascade elimina i record, ma bisogna evitare che rimangano file fisici orfani in `/uploads`.

Prima di eliminare la famiglia, raccogli in modo sicuro i file collegati a quella famiglia:

* chatMessages.fileUrl;
* billAttachments.fileUrl;
* eventuali avatar famiglia/utente se salvati in uploads;
* altri allegati realmente presenti nello schema.

Poi elimina i file fisici dal filesystem con una funzione sicura:

* solo dentro la directory uploads;
* niente path traversal;
* ignora file già mancanti;
* logga errori senza esporre path sensibili all’utente.

Attenzione:

* se la famiglia continua a esistere con altri membri, non cancellare gli allegati condivisi;
* in quel caso i contenuti possono restare visibili agli altri membri in forma anonima.

4. Rate limit su DELETE /api/auth/account

Aggiungi un rate limiter all’endpoint di eliminazione account, per evitare tentativi ripetuti sulla password.

Esempio:

* 5 tentativi ogni 15 minuti;
* disattivabile in test come per password reset.
