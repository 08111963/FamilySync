Voglio verificare e completare la gestione password in FamilySync.

Controlla prima cosa esiste già nel codice, poi proponi/implementa le parti mancanti.

Aree da verificare:

1. Password dimenticata
   Controlla se esistono già:

* endpoint richiesta reset password;
* endpoint conferma reset password;
* token reset password;
* scadenza token;
* token monouso;
* email reset password;
* schermata “Password dimenticata?”;
* schermata “Imposta nuova password”.

File da controllare:

* server/routes/auth.ts
* server/lib/email.ts
* shared/schema.ts
* app/login.tsx
* app/forgot-password.tsx
* app/reset-password/[token].tsx
* eventuali componenti auth.

Flusso richiesto:

* nella schermata login deve esserci “Password dimenticata?”;
* l’utente inserisce la propria email;
* il backend genera token sicuro;
* il token deve avere scadenza;
* il token deve essere monouso;
* meglio salvare hash del token, non token in chiaro;
* l’email deve contenere solo link sicuro, mai password;
* il link apre una schermata dove l’utente inserisce nuova password e conferma password;
* dopo reset riuscito, il token non deve più essere riutilizzabile;
* la nuova password deve rispettare le regole già usate nell’app;
* risposta generica per evitare enumeration, tipo: “Se l’email esiste, riceverai un link”.

2. Cambio password da account loggato
   Controlla se esiste già:

* endpoint cambio password;
* schermata in Profilo/Impostazioni;
* validazione password attuale;
* conferma nuova password.

Flusso richiesto:

* utente loggato apre Impostazioni/Profilo;
* clicca “Cambia password”;
* inserisce password attuale;
* inserisce nuova password;
* conferma nuova password;
* backend verifica che la password attuale sia corretta;
* se corretta, salva hash della nuova password;
* se sbagliata, risponde errore chiaro;
* non loggare password;
* non restituire mai password nelle risposte API.

3. Requisiti sicurezza

* Nessuna password via email.
* Nessuna password nei log.
* Nessuna password nelle risposte API.
* Token reset con scadenza.
* Token reset monouso.
* Preferibilmente token hashato nel database.
* Rate limit su forgot-password e reset-password.
* Password minima coerente con signup: almeno 8 caratteri, maiuscola, minuscola e numero.
* Dopo cambio password o reset, valutare se invalidare refresh token/sessioni esistenti, se il sistema lo supporta.
* In produzione, se SendGrid non è configurato, il reset password deve fallire con errore chiaro EMAIL_NOT_CONFIGURED o equivalente.

4. Email
   Usare il sistema email già presente:

* sendPasswordResetEmail;
* SENDGRID_API_KEY;
* EMAIL_FROM;
* CLIENT_URL.

Non inserire chiavi nel codice.
Le chiavi devono stare solo nei Secrets/Environment variables.

5. Frontend richiesto
   Aggiungere o verificare:

* link “Password dimenticata?” nella schermata login;
* schermata forgot-password;
* schermata reset-password con token;
* schermata cambia password dentro Profilo/Impostazioni;
* messaggi chiari di successo/errore;
* validazione nuova password;
* conferma password uguale.

6. Test richiesti
   Aggiungi test per:

* forgot password con email esistente;
* forgot password con email inesistente, risposta generica;
* reset con token valido;
* reset con token scaduto;
* reset con token già usato;
* reset con password debole;
* cambio password con password attuale corretta;
* cambio password con password attuale sbagliata;
* nessuna password viene restituita nelle risposte;
* email reset contiene link, non password.

Alla fine dammi un riepilogo:

* cosa esisteva già;
* cosa hai aggiunto;
* file modificati;
* test eseguiti;
* eventuali variabili env necessarie.
