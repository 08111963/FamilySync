========================================================

Output JSON obbligatorio:
{
  "title": string,
  "items": [
    {
      "date": "YYYY-MM-DD",
      "mealType": "breakfast"|"lunch"|"dinner"|"snack",
      "title": string,
      "description": string,
      "servings": number,
      "ingredients": [
        { "name": string, "quantity": number|null, "unit": string|null }

    }
  ]
}

Regole prompt AI:
- Ingredients sempre presenti (>=5 per pasto, salvo snack >=2)
- No brand
- Nomi ingredienti generici e puliti (es. “pomodori”, “pasta”, “petto di pollo”)
- response_format json_object + Zod validation
- fallback: se parse fallisce, return { title:"", items:[] } (senza crash)

2.2 server/routes/ai.ts

========================================================
4) ENDPOINT CONVERSIONE: USARE INGREDIENTI INLINE (E RECIPE SE PRESENTE)
========================================================
In server/routes/meal-plans.ts, endpoint convert-to-shopping-list (esistente):
- Carica meal plan items + relative ingredients:
  - Se item.recipeId presente: carica ingredienti da recipe_ingredients e includili
  - Se item.ingredients presente: includili
- Aggrega ingredienti:
  - dedupe per normalizedName
  - se quantity numerica e unit uguale: somma
  - altrimenti mantieni un’unica entry (senza somma) e lascia quantity null (o conserva la prima)
- Prima di inserire in shopping_items, filtra via:
  A) già in lista: shopping_items checked=false (famiglia) con normalizeItemName(name)
  B) spesa fatta recente: shopping_items checked=true ultimi 30 giorni (usa updatedAt se esiste, altrimenti createdAt)
- Crea o seleziona shopping list:
  - se body.shoppingListId presente -> usa quella
  - altrimenti crea una lista: "Spesa Piano <weekStartDate>" (o "<YYYY-MM-DD>")
- Inserisci shopping_items (checked=false):
  - name: string display ben leggibile, es.:
    - se quantity+unit: "Latte 2 L"
    - se solo name: "Latte"
- Risposta JSON:
  { shoppingListId, createdItemsCount, skippedAlreadyInList, skippedCompletedRecently }
