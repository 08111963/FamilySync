1) BACKEND: GENERAZIONE RICETTE PIÙ NUMEROSE E DIVERSE
========================================================

1.1 server/lib/openai.ts — generateRecipeSuggestions
- Aggiorna la funzione per accettare parametri:
  generateRecipeSuggestions({
    familySize,
    dietaryPreferences,     // string[] | string
    allergies,              // string[] | string
    maxTimeMinutes,         // number | null
    cuisinePreferences,     // string[] | null
    excludedIngredients,    // string[] | null
    lastRecipeTitles,       // string[] (per evitare ripetizioni tra run)
    count                  // number (default 12)
  })

- Output obbligatorio (JSON OGGETTO con array):
  {
    "recipes": [
      {
        "title": "...",
        "description": "...",
        "servings": 4,
        "prepTimeMinutes": 10,
        "cookTimeMinutes": 20,
        "tags": { "diet":[], "allergens":[], "cuisine":"", "difficulty":"" },
        "ingredients":[ { "name":"", "quantity": 0.0, "unit":"g", "notes":"" } ],
        "steps":[ "..." ]
      }
    ]
  }

- Requisiti “diversità” (da imporre nel prompt):
  - Generare ESATTAMENTE {count} ricette.
  - Tutte con TITOLI diversi e idee diverse (non varianti minime).
  - Vietato ripetere titoli presenti in lastRecipeTitles.
  - Varietà: alternare proteine/vegetariano, tipologie (pasta/riso/zuppa/forno/insalata), tecniche (padella/forno/vapore).
  - No brand.

- Impostazioni call:
  - response_format: json_object
  - temperature ~0.8
  - presence_penalty ~0.9
  - frequency_penalty ~0.4
- Validazione Zod: se fallisce return { recipes: [] }.

2) BACKEND: SALVATAGGIO SELETTIVO DAL PREVIEW
========================================================
In server/routes/recipes.ts:
- Aggiungi endpoint:
  POST /api/recipes/bulk
  body: {
    familyId: string,
    recipes: Array<{
      title: string,
      description?: string,
      servings?: number,
      prepTimeMinutes?: number,
      cookTimeMinutes?: number,
      tags?: any,
      ingredients: Array<{ name: string, quantity?: number, unit?: string, notes?: string }>,
      steps: string[]
    }>
  }
- Auth + membership required.
- Validazione Zod forte (no empty title, steps non vuoto, ingredients non vuoto).
- Inserisci in transaction:
  - recipes (source='ai', createdByUserId=currentUserId)
  - recipe_ingredients (normalizedName=normalizeItemName(name))
- Risposta: array di recipeIds creati.

NOTA: Non serve schema DB nuovo; usi le tabelle recipes e recipe_ingredients già presenti.
