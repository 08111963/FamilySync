1) SPESA: QUANTITÀ + UNITÀ + CATEGORIA (MVP VERO)
========================
REQUISITO
- Aggiunta item con:
  - name
  - quantity (numeric opzionale)
  - unit (enum opzionale): pcs, g, kg, ml, l
  - category (enum): food / household_cleaning / personal_care (default food)
- In lista: mostra “Latte — 2 L” e badge categoria.
- Non salvare unit dentro quantity string: unit DEVE essere campo separato.

DB (shared/schema.ts)
- shopping_items:
  - quantity numeric NULL
  - unit text NULL
  - category text NOT NULL default 'food'
  - createdByUserId uuid NULL (legacy safe)
  - createdAt timestamp default now (se manca)
- db:push (drizzle kit)

BACKEND (server/routes/shopping.ts)
- Zod:
  - category in ["food","household_cleaning","personal_care"]
  - unit in ["pcs","g","kg","ml","l"] OR null
  - quantity number >= 0 OR null
- Create item:
  - salva quantity/unit/category separati
  - createdByUserId=currentUserId
- List items:
  - include quantity/unit/category
  - applica block filtering null-safe (vedi sezione 4)
- Update item:
  - consenti update quantity/unit/category

FRONTEND (app/shopping-list.tsx)
- Form add:
  - input quantity numerico (opzionale)
  - picker unit (opzionale)
  - picker category (default food)
- Submit:
  - invia payload separato: { name, quantity, unit, category }
- Render:
  - se quantity/unit presenti: mostra “— {quantity} {unit}”
  - badge categoria

COMPATIBILITÀ DATI ESISTENTI:
- Se esistono record vecchi con quantity come stringa:
  - nel backend, quando leggi list items, se quantity non è numeric e unit è null:
    - prova parsing "2 L" -> quantity=2 unit="l" (solo in risposta, non serve migrare subito)
  - non bloccare la UI se dati legacy non parsabili.

4) BLOCCO UTENTE: FILTRARE CONTENUTI (SERVER-SIDE, NULL-SAFE) + MICRO-MIGRAZIONE
========================
OBIETTIVO
Bloccare un utente deve nascondere i suoi contenuti (eventi/spesa/chores).

BACKEND (server/lib/block-filter.ts)
Implementare helper:
- getBlockedUserIds(db, familyId, blockerUserId) -> string[]
- applyBlockedFilter(query, createdByColumn, blockedIds):
  - se blockedIds vuoto: return query
  - applica condizione:
    (createdByColumn IS NULL) OR (createdByColumn NOT IN blockedIds)

Usare applyBlockedFilter in:
- server/routes/calendar.ts (GET list eventi)
- server/routes/shopping.ts (GET list items)
- server/routes/chores.ts (GET list chores)

MICRO-MIGRAZIONE / AUDIT LEGACY (NO ATTRIBUZIONI ARBITRARIE):
