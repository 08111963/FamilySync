OBIETTIVO
Creare una “Guida Utente” completa e dettagliata per l’app Family Sync (Family-Sync-Coordination), basata SOLO su feature realmente presenti nel codice. Output in italiano, pronta per web e in-app.

VINCOLI IMPORTANTI
- NON attivare Stripe/pagamenti. Se esistono schermate premium, descriverle come “in arrivo / non disponibile” quando PREMIUM_PAYMENTS_ENABLED=false.
- Niente feature inventate: prima fare inventario leggendo il codice.
- Nessun dettaglio tecnico (endpoint/tabelle) nella guida utente.

DOVE SALVARE L’OUTPUT (OBBLIGATORIO)
1) File Markdown: /docs/guida-utente.md
2) Route pubblica web: GET /help/user-guide che serve HTML del contenuto (da markdown o HTML semplice), senza login.
3) Schermata frontend: app/help/user-guide.tsx, link in Settings -> “Guida Utente”, mostra contenuto (Markdown renderer se esiste, altrimenti WebView verso /help/user-guide).

STEP 1 — INVENTARIO FUNZIONALITÀ (PRIMA DI SCRIVERE)
Analizza i file chiave e produci una lista di feature ESISTENTI (con riferimenti ai file, ma questi riferimenti non vanno nella guida):
- Auth: signup/login/logout, refresh token, verifica email, reset password (se presenti)
- Famiglie: crea famiglia, inviti, join, ruoli (admin/member), cambio famiglia attiva
- Calendario: lista/crea/modifica/elimina eventi
- Spesa: liste, items, spuntare “comprato”, storico spesa fatta (se presente), suggerimenti AI (se presenti)
- Chores: crea/assegna/completa, punti, storico completamenti
- AI: AI insights + tab, toggle AI enabled, errore AI_DISABLED
- Moderazione UGC: segnalazioni/blocchi/pannello admin (se presenti)
- Realtime WebSocket (se presente)
- Impostazioni: privacy, termini, guida, ecc.
Qualsiasi feature non completa o dietro flag va etichettata “Disponibile prossimamente” nella guida.

0) Guida rapida (1 minuto)
1) Introduzione + a cosa serve l’app
2) Requisiti (account, connessione, permessi)
3) Creare account e accedere
4) Famiglie
5) Mini “Mappa schermate”  <-- NUOVO (qui, subito dopo le basi)

13) FAQ (almeno 15 domande)
14) Risoluzione problemi (troubleshooting)
15) Supporto e contatti
16) Changelog breve (versione attuale)

STEP 3 — CONTENUTI DELLE SEZIONI SPECIALI (OBBLIGATORIO)

A) “Guida rapida (1 minuto)”
- 6–10 passi numerati, pratici.
- Include box “Se qualcosa non funziona” che rimanda a Troubleshooting.

B) “Mini Mappa schermate” (tabella scansionabile)  <-- NUOVO

  | Schermata | Cosa trovi |
- Deve includere SOLO schermate reali, scoperte dall’inventario (esempi: Home/Calendario/Spesa/Faccende/Famiglia/AI Insights/Impostazioni/Privacy/Termini/Guida).
- Ogni riga: descrizione breve (max 1–2 frasi) di cosa si può fare.
- Se una schermata contiene funzioni disabilitate (es. Premium), indicare “in arrivo”.

Esempio stile (NON fissare contenuti se non esistono):
| Schermata | Cosa trovi |

C) “Checklist per Admin famiglia”
- Markdown task list divisa per aree:
  Setup iniziale, gestione quotidiana, sicurezza/moderazione, privacy/AI, manutenzione.
- Ogni voce deve essere concreta e basata su feature ESISTENTI.
